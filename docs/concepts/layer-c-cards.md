# Layer C — Cards

Layer C writes the prose a reader actually sees: a short, **fully cited** summary card
for each shelf and theme. Where Layers A and B are structure, Layer C is the
human-facing explanation — and it is held to a strict grounding rule.

## Every claim cites a chunk

A card is generated by an LLM, but it is **not** free-form. The model is given the
shelf's or theme's member chunks and must ground every statement in them; the card
records exactly which chunks it cited. Nothing in a card is allowed to come from the
model's parametric memory.

```python
class Card(BaseModel):
    card_id: CardId
    target_id: str                  # the shelf_id or theme_id this card describes
    target_type: Literal["shelf", "theme"]
    title: str
    summary: str
    tip: str | None
    evidence_quality: EvidenceQuality      # high | medium | low | debated | unclear
    controversy_note: str | None           # when sources disagree
    confidence_note: str | None
    cited_chunk_ids: list[str]              # provenance for every claim
    llm_model: str
    prompt_version: str
    safety_flagged: bool                    # gate for unsafe/medical-overreach output
    generated_at: datetime
```

## Honest about evidence

Nutrition literature disagrees with itself constantly, so a card carries metadata that
keeps it honest rather than authoritative:

- **`evidence_quality`** grades the strength of the underlying chunks (`high` →
  `unclear`).
- **`controversy_note`** surfaces genuine disagreement instead of averaging it away.
- **`safety_flagged`** marks output that strays into unsafe or medical-overreach
  territory so it can be withheld or reviewed.

This is what lets a downstream answer say *"the evidence is debated, and here are the
sources on both sides"* instead of inventing a confident consensus.

## Where cards plug in

Cards hang off the graph next to what they describe, and the retrieval path can fold a
card's `evidence_quality` into ranking:

```python
fs.build_layer_c()
fs.graph.shelf("foodon:olive_oil").card().summary
fs.graph.theme("...").card().cited_chunks()      # back to the source chunks
```

Because the card cites chunks and the chunks carry full provenance (source doc,
section, FoodOn IDs), the trail **card → chunk → source** is auditable end to end.
